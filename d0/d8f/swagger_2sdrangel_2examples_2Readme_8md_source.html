86 <h2>ptt_active.py</h2>
87 
88 Handles the switchover between two arbitrary device sets
89  - Both device sets should have the reverse API feature set with the address and port of this server
90  - Once in place and you have started one of the devices you should only stop one or the other never start. There are two reasons for this:
91  - This module reacts on an action already taken so if you start Tx then the Rx is not stopped immediately and damage to the Rx could occur. If you start with a stop action you cannot get in this situation.
92  - For half duplex devices (only the HackRF) it will lock Tx or Rx. You can always recover the situation by stopping the running side.
93  - There is no assumption on the Rx or Tx nature you may as well switchover 2 Rx or 2 Tx
94  - Both devices have not to belong to the same physical device necessarily. You could mix a RTL-SDR Rx and a HackRF Tx for example
95 
96 Depends on `flask` and `requests`: to install do `pip install flask requests` in your virtual environment.

136 <h2>reverseapi.py</h2>
137 
138 This script runs continuously to act as the server side of SDRangel reverse API feature. It will respond with a copy of the settings when forwarding channel or device settings which is a valid reply and will also send back a valid reply when forwarding the device start/stop actions. The reverse API feature was implemented in version 4.3.2 and you can learn more about it in the Wiki.
139 
140 It uses Python flask package for the server so you will need to either install it system wide or better create your own virtual environment and use the `pip install flask` command once your virtual environment is activated. If you know Python then you know what a virtual environment is about.
141 
142 Then in the virtual environment do:
143 
144 ```
145 export FLASK_APP=reverseapi.py
146 export FLASK_ENV=development
147 flask run
148 ```
149 By default the server will listen on port 5000. It was used to conveniently test the reverse API but actually does not do much. It can serve as a base to implement more complex scenarios to control other devices or software or other instances of SDRangel.

221 <h2>scanner.py</h2>
222 
223 Simple AM and NFM scanner with multiple equally spaced channels. Stops whenever any of the channels squelch opens. At the moment the following sampling devices can be used:
224 
225  - AirspyHF
226  - HackRF
227  - LimeSDR
228  - RTLSDR
229 
230 Check `./scanner.py --help` for the options.
231 
232 Requires numpy
233 
234 &#9758; Try to first run with the `-m` (mock) option to see what the scanner parameters will be. In particular pay attention to the computed sample rate that should be within range for your hardware. For devices with fixed or discrete rates make sure the computed sample rate does not exceed the unique device sample rate or the highest possible sample rate.
